Como utilizar o Git/GitHub

GIT: sistema versionamento de arquivos, que permite ter controle de tudo que já foi realizado ou alterado em um arquivo. Em que várias pesso as podem modificar o mesmo arquivo de diferentes dispositivos.

GIT HUB: é uma plataforma para hospedar arquivos que estão sendo utilizados no GIT.

  • REPOSITÓRIOS: são os diretórios do GIT para armazenar projetos, os códigos podem ser compartilhados para outras pessoas. Muito usado para trabalhos em equipe. Extremamente importante no mercado de trabalho, principalmente para área de computação (desenvolvimento), podem ser adicionadas novas versões a partir de um post.

Status: mostra o que está acontecendo no projeto. Ele mostra uma serie de informações como a branch atual, o último commit e branches remotos, estado do repositório, etc.

Add: salva e adiciona arquivos ao fluxo de versionamento, para serem incluídos no próximo commit.

  • git add . → Adiciona todos os arquivos do diretório
  • git add nomearquivo → Adiciona um arquivo desejado.
  • git add nomediretório → adiciona todos do arquivos de um diretório

    • Commit: usado para confirmar as alterações em um repositório.

  • git commit → salva alterações.
  • git commit -m →permite uma mensagem no commit, importante para comunicação em equipe.

    • Branch: usado para mostrar os branchs (ramificações) ativos do projeto, além de poder criar e excluir branchs.

  • git branch nomebranch → cria um novo branch desejado a partir do commit atual.
  • git branch -d nomebranch → exclui uma branch, que esteja unido a outra.
  • git branch -D nomebranch → exclui uma branch, mesmo que não esteja unida a outra.
  • git branch -m nomebranch → renomeia a branch.
  • git branch -r → lista todas as branches remotas.
  • git branch -a → lista todas as branches locais e remotas.
    • Obs: o branch principal é main do projeto.

    Push: envia as alterações de um branch para um repositório remoto, atualizando o repositório remoto. Antes de usar o push, deve ser feito um commit préviamente.

    Checkout: sado para alterna de branches de diferentes maneiras.

  • git checkout nomebranch → muda para uma branch já criada.
  • git checkout -b nomebranch → cria uma branch, já alternando para ela.
  • git checkout - - nomebranch→ descarta mudanças de um arquivo.
  • git checkout - - . → descarta mudanças depois do último commit.

    • Push: envia as alterações de um branch local para um repositório remoto, atualizando o repositório remoto. Antes de usar o push, deve ser feito um commit préviamente.

    Checkout: usado para alterna de branches de diferentes maneiras.

  • git checkout nomebranch → muda para uma branch já criada.
  • git checkout -b nomebranch → cria uma branch, já alternando para ela.
  • git checkout - - nomebranch→ descarta mudanças de um arquivo.
  • git checkout - - . → descarta mudanças depois do último commit.

    • Merge: une os códigos, incorpora as alterações de um branch para outro.

  • Git merge nomebranch → une uma branch com a atual.
    • Obs: caso as mesmas linhas dos branches forem modificadas, poderá ocorrer conflitos no merge, que devem ser corrigidos manualmente.

    Pull: coloca as atualizações do remoto no repositório local. Possibilita o acesso as mudança recentes no código.

  • git pull nomeremoto nomebranch → une o branch remoto com o atual.

    • Clone: clona e manda projetos para o repositório atual. Baixa os arquivos do repositório remoto e cria um diretório local com uma cópia do projeto.

  • git clone URLremoto nomediretorio → clona o repositório em um diretório.

    • Init: usado para criar um novo repositório GIT em um diretório local vazio. Obs: o comando não tem efeito se o diretório já for GIT.

    Log: explica cada mudança que o projeto teve, mostrando os commits realizados, só funciona para arquivos já commitados.

  • git log → mostra os commits dos arquivos com data, hora que foram executados.
  • git log nomearquivo → mostra informações sobre um arquivo desejado.
  • git log - -oneline → mostra apenas a primeira linha do commit, deixando a visualização mais fácil.
  • git log - -author=nomeautor → visualiza o histórico de commit de uma pessoa.
  • git log - -since=datacommit → visualiza commit de uma determinada data.

    • Rm: remove arquivos e diretórios.

  • rm nomearquivo → remove um arquivo.
  • rm -r nomediretorio → remove um diretório incluindo todos os arquivos armazenados nele.

    • Reset: usado para desfazer uma alteração local de algumas formas. Ele apaga os commits, reescrevendo a linha do tempo, portanto é recomendado que utilize-o no repositório local, antes de ser realizado um push.

  • git reset nomecommit → redefine o branch atual para o commit desejado, descartando commits posteriores.
  • git reset - -hard nomecommit → também redefine o branch, mas alterações não salvas são permanentemente perdidas.
  • git reset - -soft nomecommit → redefine o branch como os outros, entretanto mantém arquivos no diretório, que podem ser refeitas em um novo commit.

    • Revert: diferente do reset, ele desfaz alterações criando um novo commit pelo commit original, desse modo ele não reescreve a linha do tempo dos commits.

  • git revert nomecommit → desfaz as mudanças colocando-as no novo commit.
  • git commit HEAD → desfaz as mudanças do commit mais recente.
  • git revert nomecommit1 nomecommit2 → indica a linha do tempo que serão criados novos commits com para desfazer as mudanças.

    • Tag: são usados para marcar pontos específicos na linha do tempo do repositório, assim definem as versão do que está sendo realizado no projeto. Importante para códigos grandes para que facilite a compreensão.

  • git tag → lista as tags do repositório.
  • git tag nometag → cria uma tag.
  • git tag -a nometag -m “mensagem” → cria uma tag com uma mensagem associada a ela.
  • git tag -d nometag → exclui uma tag.
  • git show nometag → mostra informações sobre a tag.

    • Rebase: serve para reescrever / reorganizar o histórico do repositório, ele muda a posição dos commits na linha de tempo de diversas formas. Importante para deixar o histórico mais linear de uma forma que facilite o trabalho.

  • Git rebase nomebranch → aplica os commits do branch atual para outro branch na ponta da linha de tempo.
  • Git -i nomecommit → lista os commits em um editor de texto, que podem ser reordenados de acordo com o desejo do usuário.

    • Blame: permite analisar quem foi o autor, horário do último commit e outras informações associadas a uma linha de código, comando bom para ser usado em códigos grandes, realizados em equipe, visto ao identificar quem realizou um commit, melhora a comunicação direta. Git blame nomearquivo → mostra informações de alterações feitas linha por linha.
